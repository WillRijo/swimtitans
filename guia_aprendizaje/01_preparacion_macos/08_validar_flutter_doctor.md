# Validar Flutter doctor

**Nivel:** Practicar

## Objetivo

Usar `flutter doctor` para revisar si Flutter esta listo.

## Herramientas que vamos a usar

- VS Code
- Terminal integrada
- Flutter
- Comando `flutter doctor`

## Idea principal

`flutter doctor` es como una revision medica para Flutter.

Revisa si la computadora tiene lo necesario para crear y correr apps.

Puede mostrar:

- cosas correctas
- cosas que faltan
- advertencias
- errores

## Paso 1: Abrir VS Code

1. Abre VS Code.
2. Espera a que cargue.

## Paso 2: Abrir la terminal integrada

1. Ve al menu **Terminal**.
2. Elige **New Terminal**.

Tambien puedes usar `Control + `` si funciona en tu teclado.

## Paso 3: Ejecutar Flutter Doctor

En la terminal integrada, escribe este comando:

```bash
flutter doctor
```

Presiona `Enter`.

Espera el resultado.

## Paso 4: Leer el resultado

Si ves checks o mensajes positivos, eso significa que algo esta bien.

Si ves `warning` o `error`, significa que falta algo o hay que revisar.

No siempre hay que arreglar todo de inmediato.

Para empezar con web, normalmente lo mas importante es que Flutter y Chrome esten listos.

Para iPhone o Android pueden aparecer pasos extra.

## Paso 5: Copiar el resultado si hace falta

Si no entiendes el resultado, copia el texto.

Guardalo en una nota.

Pide ayuda con ese mensaje.

## Si algo sale mal

- Si dice que `flutter` no existe, cierra y abre VS Code otra vez.
- Si sigue igual, puede faltar agregar Flutter al `PATH`.
- Si aparece un mensaje largo, no te asustes.
- Copia el mensaje antes de cerrar la terminal.

## Lo que haremos despues

En la siguiente leccion crearemos el primer proyecto Flutter vacio.

## Actividad

1. Abre VS Code.
2. Abre la terminal integrada.
3. Ejecuta:

```bash
flutter doctor
```

4. Anota una cosa que salio bien.
5. Anota una cosa que no entiendas, si aparece.
